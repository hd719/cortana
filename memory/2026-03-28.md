## ACP troubleshooting — Telegram/Codex session failures
- Investigated repeated ACP Codex run failures where sessions spawned but returned no messages/commits and timed out.
- `openclaw doctor` showed two likely configuration issues:
  1. `plugins.entries.acpx` duplicate plugin id override (configured acpx plugin overriding bundled acpx plugin).
  2. `channels.telegram.threadBindings` is invalid/unrecognized in current build config schema.
- Context note: this conversation was in Telegram DM (`chat_type: direct`), so ACP thread-binding behavior is not needed for this path.
- Recommended remediation sequence: remove explicit duplicate `plugins.entries.acpx` override, remove/relocate invalid `channels.telegram.threadBindings`, restart gateway, then run ACP smoke test with `sessions_spawn` using `runtime:"acp", mode:"run", thread:false`.
