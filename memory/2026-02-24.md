## Key Decisions
- Canonicalized strict protocol contracts for agent spawning and completion payloads (validation-first orchestration).
- Added remediation guardrails to prevent runaway self-healing loops (cooldown + daily caps).
- Treated Cortana DB as preferred task source of truth, with automatic reconciliation to app DB to eliminate stale board states.
- Required verification gates for automation that mutates task status (post-merge autoclose must verify done state or emit critical event).
- Separated health scoring into explicit computation logic (`agent-health`) to keep scoring responsive for both new and mature agents.
