# Proactive Signal Precision Calibration

This module audits proactive alert quality and tunes category thresholds to reduce noise.

## Files

- `tools/proactive/signal_calibrator.py`
- `config/alert-thresholds.json`
- `reports/proactive-signal-audit.json` (generated by `audit`)

## What it measures

For a lookback window (`--days`, default 30), the calibrator:

1. Pulls proactive alert-like events from:
   - `cortana_proactive_signals` (primary)
   - `cortana_events` (fallback/supplement)
2. Classifies each alert into one category:
   - `portfolio`, `email`, `calendar`, `weather`, `health`, `tech_news`
3. Cross-references `cortana_feedback` for noise markers (`noise`, `spam`, `alert fatigue`, etc.).
4. Checks action outcomes:
   - Task creation linked by timing/text within 30 minutes (`cortana_tasks`)
   - User response signals within 30 minutes (`cortana_events` reply/message-style records)
5. Computes precision:
   - `alerts_that_led_to_action / total_alerts`

## CLI usage

### Audit

```bash
python3 tools/proactive/signal_calibrator.py audit --days 30
```

Output:
- JSON summary to stdout
- Saved snapshot at `reports/proactive-signal-audit.json`

### Report

```bash
python3 tools/proactive/signal_calibrator.py report
```

Human-readable summary includes:
- Overall precision
- Per-category precision
- Top noise sources (high volume + low action)
- Threshold recommendations

### Tune

```bash
python3 tools/proactive/signal_calibrator.py tune --category health --threshold 0.75
```

- Updates `config/alert-thresholds.json`
- Valid category list is enforced
- Threshold must be in `[0,1]`

## Threshold config

`config/alert-thresholds.json` stores tunable values per category:

```json
{
  "portfolio": 0.7,
  "email": 0.65,
  "calendar": 0.6,
  "weather": 0.55,
  "health": 0.65,
  "tech_news": 0.6
}
```

## Notes

- Correlation is intentionally conservative: the script requires temporal proximity (30 minutes) and matching text/category clues.
- If your environment logs richer user-response metadata, precision will become more accurate automatically.
- Recommendations are generated from current precision + threshold values, but tuning decisions should still be reviewed by a human.
